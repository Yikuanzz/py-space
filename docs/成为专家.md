### Python 基础知识

作为初学者，首先要打牢基础。这些是 Python 专家必须熟练掌握的核心元素，没有它们就无法构建更复杂的应用。

- **语法和基本概念**：熟练使用变量、数据类型（int, float, str, bool 等）、运算符、表达式。理解 Python 的动态类型系统和内存管理（如垃圾回收）。
- **控制流**：if-else、循环（for, while）、break/continue/pass。能够编写高效的条件判断和迭代逻辑。
- **数据结构**：内置结构如列表（list）、元组（tuple）、字典（dict）、集合（set）。掌握切片（slicing）、列表推导式（list comprehensions）、字典推导式等高级用法。
- **函数**：定义函数（def）、参数（位置参数、关键字参数、可变参数 *args/**kwargs）、返回值、lambda 匿名函数。理解作用域（LEGB 规则）和闭包（closures）。
- **模块和包**：import 语句、创建自定义模块、理解 __init__.py 和包结构。熟练使用 pip 安装和管理第三方包。

### 面向对象编程 (OOP)

Python 是多范式语言，OOP 是专家级开发者的必备技能。

- **类和对象**：定义类（class）、实例化对象、属性（instance/class variables）、方法（instance/class/static methods）。
- **继承和多态**：单继承、多继承（菱形继承问题）、super() 函数、重写方法、多态实现。
- **特殊方法**：`__init__`、`__str__`、`__repr__`、`__len__` 等魔法方法（dunder methods），用于自定义类的行为。
- **封装和抽象**：私有属性（_ 或 __ 前缀）、抽象类（使用 abc 模块）。

### 错误处理和调试

专家需要能优雅地处理问题，而不是让程序崩溃。

- **异常处理**：try-except-else-finally、自定义异常、raise 语句。理解常见异常如 ValueError、IndexError。
- **调试工具**：使用 pdb 模块、print 调试、IDE 如 VS Code 或 PyCharm 的调试器。熟练阅读 traceback。

### 文件和 I/O 操作

实际应用中，数据读写是常态。

- **文件操作**：open() 函数、读写模式（r/w/a）、with 语句上下文管理器。处理文本文件、CSV、JSON 等格式。
- **序列化**：json 模块、pickle 模块，用于数据持久化。

### 标准库

Python 的强大在于其标准库，专家应熟悉常用模块，避免重复造轮子。

- **内置模块**：os/sys/pathlib（文件系统操作）、datetime/time（时间处理）、math/random（数学和随机）、re（正则表达式）、collections（高级数据结构如 deque, Counter）。
- **网络和 web**：requests（HTTP 请求）、http.server（简单服务器）、socket（底层网络）。
- **多线程/进程**：threading/multiprocessing 模块，理解 GIL（全局解释器锁）的限制。

### 第三方库和生态

Python 生态丰富，专家需根据领域选择性掌握，但以下是通用热门库：

- **数据科学和分析**：numpy（数组操作）、pandas（数据帧处理）、matplotlib/seaborn（可视化）、scipy（科学计算）。
- **机器学习**：scikit-learn（经典 ML）、tensorflow/pytorch（深度学习）。
- **Web 开发**：flask/django（web 框架）、fastapi（异步 API）、beautifulsoup（网页抓取）。
- **自动化和脚本**：selenium（浏览器自动化）、openpyxl（Excel 处理）、pillow（图像处理）。
- **数据库**：sqlite3（内置）、SQLAlchemy（ORM）、pymongo（MongoDB）。

### 高级主题

成为专家后，需要处理复杂场景。

- **异步编程**：asyncio 模块、async/await、coroutines。用于高并发 I/O 操作，如网络应用。
- **并发和并行**：多线程 vs 多进程 vs 异步，理解线程安全、锁（Lock）、队列（Queue）。
- **性能优化**：使用 cProfile/timeit 分析性能、JIT（如 Numba）、C 扩展（ctypes/cffi）。
- **测试和质量**：unittest/pytest（单元测试）、mock（模拟对象）、黑盒/白盒测试。遵循 PEP 8 代码风格，使用 black/flake8 工具。
- **部署和 DevOps**：virtualenv/conda（环境管理）、docker（容器化）、CI/CD（如 GitHub Actions）。
